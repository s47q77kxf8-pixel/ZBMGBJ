# 报价小票显示规则

## 一、小票整体结构

```
┌─────────────────────────────────────────────────────────────┐
│ 顶部信息区（日期、单主、画师、IP、角色）                      │
├─────────────────────────────────────────────────────────────┤
│ 制品详情区（表格：制品、单价、数量、小计）                    │
├─────────────────────────────────────────────────────────────┤
│ 赠品信息区（如有赠品）                                       │
├─────────────────────────────────────────────────────────────┤
│ 底部汇总区（加价、折扣、其他费用、应收、实付）                │
├─────────────────────────────────────────────────────────────┤
│ 结算信息区（如有结算）                                       │
└─────────────────────────────────────────────────────────────┘
```

## 三、制品详情区

### 3.1 列结构

| 列名 | 宽度 | 说明 |
|------|------|------|
| 制品 | 2列 | 制品名称、明细行 |
| 单价 | 1列 | 单价金额 |
| 数量 | 1列 | 数量（件数） |
| 小计 | 1列 | 总价金额 |

### 3.2 制品类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `fixed` | 固定单价制品 | 方宣、明信片 |
| `double` | 双面制品（需标注单/双面） | 方卡(单面)、方卡(双面) |
| `config` | 基础配置+配件 | 立牌（含底座、插件等） |
| `nodes` | 按节点收费 | 草稿、色稿、线稿等 |

### 3.3 合并显示规则（canMerge）

**满足以下全部条件时，合并到总览行显示：**
- 非nodes类型
- 无同模制品（`sameModelCount === 0`）
- 无工艺（`processDetails.length === 0`）
- 非config类型
- 类型为fixed，或类型为double且无额外配件

**合并显示示例：**

```
1. 方宣          ¥70.00    1件    ¥70.00
```

### 3.4 拆分显示规则

**不满足合并条件时，显示总览行+明细行：**

#### 3.4.1 fixed/double类型（有同模或工艺）

```
1. 方卡(单面)    —          4件    ¥200.00
   • 全价制品    ¥50.00     1      ¥50.00
   • 同模制品(0.5x) ¥25.00  3      ¥75.00
   • 工艺
     烫色×1      ¥10.00     4      ¥40.00
```

#### 3.4.2 config类型（基础配置+配件）

```
1. 立牌          ¥230.00   1件    ¥230.00
   └ 立牌+底座×1 —
   └ 插件×4      ¥120.00
   └ 底座×1      ¥40.00
```

**config显示逻辑：**
- 无额外配件：显示成品单价
- 有额外配件但无同模无工艺：显示成品单价（已包含配件）
- 有同模或工艺：总览行单价显示"—"，明细行显示具体单价

#### 3.4.3 nodes类型（按节点收费）

```
1. 头像委托      ¥200.00   1件    ¥200.00
   • 草稿 50%                ¥100.00
   • 线稿 30%                ¥60.00
   • 色稿 20%                ¥40.00
```

### 3.5 明细行规则

#### 3.5.1 全价制品行

```
• 全价制品    ¥50.00     1      ¥50.00
```

**计算逻辑：**
- 全价制品单价 = 基础单价 + 每制品增加费用
- 全价制品数量 = 1（有同模时）或总数量（无同模时）

#### 3.5.2 同模制品行

```
• 同模制品(0.5x)    ¥25.00     3      ¥75.00
```

**计算逻辑：**
- 同模系数显示：`minus`模式显示`−¥xx`，`rate`模式显示`0.5x`
- 同模制品单价 = 同模单价 + 每制品增加费用
- 同模制品数量 = 同模件数

#### 3.5.3 工艺行

```
• 工艺
  烫色×1、覆膜×1    ¥10.00     8      ¥80.00
```

**计算逻辑：**
- 按每层单价分组，同单价的工艺合并为一行
- 每行最多显示2个工艺
- 计费数量 = 总层数 × 件数

#### 3.5.4 基础配置行（config类型）

```
└ 立牌+底座×1    ¥70.00
```

**显示规则：**
- 基础配置价格 = 总基础价 - 配件总额
- 如果基础配置价格为0，不显示这一行
- 如果基础配置价格与全价制品单价相同，显示"—"

#### 3.5.5 配件明细行（config类型）

```
└ 插件×4        ¥120.00
└ 底座×1        ¥40.00
```

**计算逻辑：**
- 每件配件价格 = 配件单价 × 配件数量

## 四、赠品信息区

### 4.1 显示规则

```
─────────────────────────────────────────────────────────────
                        赠品信息
─────────────────────────────────────────────────────────────
[赠品] 方卡(单面)    —        4件    ¥0.00
                                     ¥215.00（划线原价）
   • 全价制品        ¥50.00   1      ¥50.00
   • 工艺
     烫色×1          ¥10.00   4      ¥40.00
```

### 4.2 赠品特殊处理

| 项目 | 规则 |
|------|------|
| 小计 | 显示`¥0.00`（免费） |
| 原价 | 使用`giftOriginalPrice`（包含工艺和额外费用），划线显示 |
| 明细 | 显示原价明细（同制品规则） |
| 大类分隔 | 不同大类之间显示分隔线 |

## 五、底部汇总区

### 5.1 显示顺序

1. 制品小计（可选）
2. 加价合计（可选）
3. 折扣合计（可选）
4. 其他费用合计（可选）
5. 应收金额（可选，含抹零优惠）
6. 平台费（可选）
7. 实付金额（必显示）
8. 定金信息（可选）

### 5.2 制品小计

```
制品小计        ¥200.00
```

**显示条件：** 制品小计 ≠ 应收金额（差值 ≥ 0.005）

### 5.3 加价合计

```
加价合计：1.2×   ¥240.00
  用途系数：1.1×
  加急系数：1.09×
```

**包含系数：**
- 用途系数（usage）
- 加急系数（urgent）
- 扩展加价系数（extraUpSelections）

**显示条件：** 加价系数 ≠ 1

### 5.4 折扣合计

```
折扣合计：0.9×   -¥24.00
  折扣系数：0.9×
```

**包含系数：**
- 折扣系数（discount）
- 扩展折扣系数（extraDownSelections）

**显示条件：** 折扣系数 ≠ 1

### 5.5 其他费用

```
其他费用合计     ¥50.00
  运费 ¥50.00
```

**显示条件：** 其他费用 > 0

### 5.6 应收金额

#### 5.6.1 无抹零

```
应收金额        ¥266.00
```

#### 5.6.2 有抹零

```
应收金额（-¥1.00）  ¥265.00
                     ¥266.00（划线原价）
```

**显示条件：**
- 约定实收 ≠ 系统计算原价（差值 ≥ 0.005）
- 或有平台费且实付 ≠ 应收

### 5.7 平台费

```
平台费 5%       +¥13.25
```

**显示条件：** 平台费 > 0

### 5.8 实付金额

```
实付金额        ¥278.25
```

**计算逻辑：**
- 有平台费：实付金额 = 应收金额 + 平台费
- 无平台费：实付金额 = 应收金额

### 5.9 定金信息

#### 5.9.1 需付定金（未收定时）

```
需付定金        ¥83.48
```

**计算：** 实付金额 × 定金比例（默认30%）

#### 5.9.2 已收定金

```
已收定金        ¥100.00
```

**显示条件：** 已收定金 > 0

## 六、结算信息区

### 6.1 显示规则

```
───────── 结算结果 ─────────
（以下为结算结果，收费/退款以撤单·废稿·结算页确认为准）

结算类型：正常结单
```

### 6.2 结算类型

| 类型 | 显示文本 |
|------|----------|
| `full_refund` | 撤单（退全款） |
| `cancel_with_fee` | 撤单（收跑单费） |
| `waste_fee` | 废稿结算 |
| `normal` | 正常结单 |

### 6.3 撤单（退全款）

```
结算类型：撤单（退全款）
本单状态：已撤销，本单作废
原已收定金：         ¥100.00
需退金额：           ¥100.00
已退款金额：         ¥100.00
结算结果：           已全额退还定金。
```

### 6.4 正常结单

```
结算类型：正常结单
企划应收：           ¥265.00
已收定金：           ¥100.00
本次结算：           ¥165.00
合计已收：           ¥265.00
结算结果：           已全额结清。
```

## 八、特殊标记

| 标记 | 说明 |
|------|------|
| `[跨单同模]` | 跨订单同模制品标记 |
| `—` | 单价留空（明细行显示具体单价） |
| 划线金额 | 原价（赠品、抹零优惠时显示） |
| `•` | 明细行项目符号 |
| `└` | config类型配件项目符号 |

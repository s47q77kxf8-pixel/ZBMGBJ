# 修复小票显示"基础配置(单面)"或"基础配置(双面)"问题

## 问题分析

当前代码中，只有当制品的 `priceType` 为 `'double'` 时，才会显示"基础配置(单面)"或"基础配置(双面)"。但根据用户需求，应该是**无论制品的价格类型是什么**，只要是单面就显示"基础配置(单面)"，双面就显示"基础配置(双面)"。

## 修复方案

修改 `generateQuote` 函数中的条件判断，去掉 `item.productType === 'double'` 的限制，改为只要制品有 `sides` 属性就显示对应的基础配置信息。

## 具体修改

在 `script.js` 文件中，找到 `generateQuote` 函数，将第1285行的条件判断：

```javascript
if (item.productType === 'double' && item.productId) {
```

修改为：

```javascript
if (item.sides) {
```

这样，无论制品的价格类型是什么，只要有 `sides` 属性（单面或双面），就会显示对应的"基础配置(单面)"或"基础配置(双面)"。

## 预期效果

修复后，小票上会根据制品的单双面属性显示对应的基础配置信息：
- 当制品为单面时，显示"• 基础配置(单面)"
- 当制品为双面时，显示"• 基础配置(双面)"
- 无论制品的价格类型是固定价、单双面价还是基础+递增价，都会显示

## 修复位置

- 文件：`script.js`
- 函数：`generateQuote`
- 行号：第1285行

## 验证方法

1. 选择任意制品（如普通吧唧、异形吧唧等）
2. 选择单双面属性
3. 完成计算并生成报价
4. 检查小票，应该会显示对应的"基础配置(单面)"或"基础配置(双面)"

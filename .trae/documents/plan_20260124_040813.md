# 修复小票基础配置显示问题

## 问题分析

之前的修复错误地显示了所有制品的基础配置，包括固定价类型，这不符合用户的要求。用户希望：

1. **固定价**：不显示基础配置
2. **单双面价**：
   - 单面：显示"• 基础配置(单面)"
   - 双面：显示"• 基础配置(双面)"
3. **基础+递增价**：按现在显示基础配置

## 解决方案

需要修改 `generateQuote` 函数中的条件判断逻辑，按照用户的要求来显示基础配置：

### 1. 制品部分修复

将当前的条件判断：
```javascript
if (item.sides === 'single') {
    // 单面制品，显示基础配置(单面)
    // ...
} else if (item.sides === 'double') {
    // 双面制品，显示基础配置(双面)
    // ...
} else if (item.productType === 'config' && item.baseConfig) {
    // 基础+递增价类型，显示基础配置信息
    // ...
}
```

修改为：
```javascript
if (item.productType === 'double') {
    // 单双面价类型
    if (item.sides === 'single') {
        // 单面，显示"• 基础配置(单面)"
        // ...
    } else if (item.sides === 'double') {
        // 双面，显示"• 基础配置(双面)"
        // ...
    }
} else if (item.productType === 'config' && item.baseConfig) {
    // 基础+递增价类型，显示基础配置信息
    // ...
}
// 固定价类型，不显示基础配置
```

### 2. 赠品部分修复

同样需要修改赠品部分的条件判断逻辑，保持与制品部分的一致性：

```javascript
if (item.productType === 'double') {
    // 单双面价类型
    if (item.sides === 'single') {
        // 单面，显示"• 基础配置(单面)"
        // ...
    } else if (item.sides === 'double') {
        // 双面，显示"• 基础配置(双面)"
        // ...
    }
} else if (item.productType === 'config' && item.baseConfig) {
    // 基础+递增价类型，显示基础配置信息
    // ...
}
// 固定价类型，不显示基础配置
```

## 修复位置

- 文件：`script.js`
- 函数：`generateQuote`
- 制品部分：第1279-1288行
- 赠品部分：第1362-1371行

## 预期效果

修复后，小票上会按照用户的要求显示基础配置：

1. **固定价**：不显示基础配置
2. **单双面价**：
   - 单面：显示"• 基础配置(单面)"
   - 双面：显示"• 基础配置(双面)"
3. **基础+递增价**：显示"• 基础配置(${item.baseConfig})"

## 修复验证

修复后，当您选择不同价格类型的制品并完成计算生成报价时，小票上会按照用户的要求显示对应的基础配置信息。
